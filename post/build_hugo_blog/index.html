用hugo建立博客

2018-01-24
introduction

傻瓜式安装Hugo

Windows

借助windows的包管理工具chocolatey - 安装 chocolatey - 在命令行输入

choco install hugo -confirm

Linux

直接从这里下载编译好的包. 解压,将bin路径配置到环境变量中.

从源码安装

安装: - git - Go 获取源码并编译:

export GOPATH=$HOME/go
go get -v github.com/spf13/hugo

生成博客站点

在你希望存储站点相关文件的位置my_blog_path执行

hugo new site [my_blog_path]

在my_blog_path会生成如下结构文件:

.
├── archetypes
├── config.toml
├── content
├── data
├── layouts
├── static
└── themes

创建第一篇文章

在my_blog_path中执行:

hugo new post/first.md

自动生成了文件content/post/first.md.

---
title: "First"
date: 2018-01-24T11:59:06+08:00
draft: true
---

正文

---之间的内容是关于文章的一些元数据,以toml格式指定. 之后可以开始写正文.

安装一个主题

Hugo的开源社区有许多好看的主题,可以在这里查看.选择一个你喜欢的,找到github地址,将整个仓库克隆到themes文件夹中.比如我们选择Hyde:

git clone https://github.com/spf13/hyde.git thems/hyde

运行Hugo

你可以在本地看看博客的效果,在my_blog_path运行:

hugo server --theme=hyde --buildDraft

hugo服务默认绑定在1313端口,你可以在浏览器中打开http://localhost:1313查看.

简单部署到Github上

你需要一个github账号,假如你的账号是littlebeandog. 在你的githb上创建一个名称为littlebeandog.github.io的仓库.

在my_blog_path路径下执行:

hugo --theme=hyde --baseUrl="http://littlebeandog.github.io/"

博客的静态页面会生成在public路径下. 将public路径中的所有内容push到刚刚创建的仓库的master分支,在public路径中执行如下命令:

git init
git remote add origin https://github.com/littlebeandog/littlebeandog.github.io.git
git add -A
git commit -m 'init'
git push -u origin master

现在任何人都可以在公网通过http://littlebeandog.github.io/访问你的博客了.
